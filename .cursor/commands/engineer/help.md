# /engineer/help

Guia rápido da trilha de engenharia no Sistema Onion.

## Fluxo principal
`/engineer/plan -> /engineer/start -> /engineer/work -> /engineer/pre-pr -> /engineer/pr -> /engineer/pr-update`

## Quando usar cada etapa
- `plan`: definir abordagem técnica e riscos.
- `start`: preparar sessão e contexto da feature.
- `work`: executar implementação por fases.
- `pre-pr`: validar qualidade antes de abrir PR.
- `pr`: criar PR com contexto e checklist.
- `pr-update`: atualizar PR com progresso e ajustes.
